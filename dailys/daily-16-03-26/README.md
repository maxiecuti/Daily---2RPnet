# Relatório de Atividades Técnicas — 16 de Março de 2026

O dia foi dedicado ao estabelecimento de uma base sólida tanto na administração do ambiente operacional quanto nos conceitos fundamentais de processamento distribuído com **Apache Spark**. A jornada iniciou-se com a configuração e exploração do ambiente **Linux via VirtualBox**, onde foi essencial dominar a interface de linha de comando para garantir a agilidade necessária no manejo de grandes volumes de dados. Através da utilização de comandos de navegação e manipulação hierárquica de diretórios, foi possível estruturar um workspace organizado, garantindo que o fluxo de ingestão e saída de arquivos respeitasse as boas práticas de governança de sistemas. No âmbito do **PySpark**, o foco principal recaiu sobre a compreensão da arquitetura de dados, especificamente na transição do modelo de **RDD** para a abstração de **DataFrames**. Este estudo permitiu uma análise profunda sobre a imutabilidade dos dados e a linhagem do Spark, diferenciando operações de **Transformação**, que utilizam a *lazy evaluation* para otimizar o plano de execução, das **Ações** que efetivamente processam os dados. Além disso, a implementação de **schemas estruturados** e o entendimento da tipagem de dados foram fundamentais para assegurar a integridade das informações desde o momento da leitura, evitando inconsistências técnicas que poderiam comprometer as etapas subsequentes de análise e integração.
